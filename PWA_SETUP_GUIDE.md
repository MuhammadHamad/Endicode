## üé® Icon Requirements

To complete the PWA setup, you need proper app icons. Currently using `/brand/endicode-emblem.png`.

### Required Icon Sizes:
- **192x192** - Standard Android icon
- **512x512** - High-res Android icon
- **180x180** - Apple Touch Icon (iOS)

### How to Generate Icons:

#### Option 1: Use Online Tool (Easiest)
1. Go to https://realfavicongenerator.net/
2. Upload your Endicode logo/emblem
3. Configure settings:
   - iOS: Use solid background color (#0a0a0a)
   - Android: Use transparent or solid background
4. Download the generated icons
5. Place them in `public/brand/` folder

#### Option 2: Manual Creation
If you have the source logo:
1. Open in image editor (Photoshop, GIMP, Figma, etc.)
2. Create versions at:
   - 192x192 pixels ‚Üí `icon-192x192.png`
   - 512x512 pixels ‚Üí `icon-512x512.png`
   - 180x180 pixels ‚Üí `apple-touch-icon.png`
3. Save as PNG with transparency (or solid background)
4. Place in `public/brand/` folder

#### Option 3: Use PWA Asset Generator (Automated)
```powershell
# Install globally
npm install -g @vite-pwa/assets-generator

# Generate icons from your logo
pwa-assets-generator --preset minimal public/brand/endicode-emblem.png
```

## üß™ Testing the PWA

### Local Testing:

1. **Build the production version:**
   ```powershell
   npm run build
   ```

2. **Serve the production build:**
   ```powershell
   npm run start
   ```

3. **Open in browser:**
   - Chrome: http://localhost:5000 (or your configured port)
   - Look for the install icon in the address bar
   - Click to install the PWA

### Testing Checklist:

- [ ] **Install Prompt**: Browser shows "Install App" button
- [ ] **Offline Mode**: Disconnect internet, app still loads
- [ ] **App Icon**: Icon appears on desktop/home screen after install
- [ ] **Standalone Mode**: Opens in its own window (no browser UI)
- [ ] **Theme Color**: App bar matches your theme color
- [ ] **Updates**: Changes deploy and update automatically

### Browser DevTools Testing:

#### Chrome DevTools:
1. Open DevTools (F12)
2. Go to **Application** tab
3. Check:
   - **Manifest**: Should show all your app details
   - **Service Workers**: Should be registered and active
   - **Cache Storage**: Should show cached assets
4. Test offline:
   - Check "Offline" in Network tab
   - Reload page - should still work

#### Lighthouse Audit:
1. Open DevTools (F12)
2. Go to **Lighthouse** tab
3. Select "Progressive Web App" category
4. Click "Generate report"
5. Aim for 90+ score

## üîß Configuration Options

### Update Manifest Colors:
Edit `public/manifest.json`:
```json
{
  "theme_color": "#your-color",
  "background_color": "#your-color"
}
```

### Change Caching Strategy:
Edit `vite.config.ts` ‚Üí `workbox` section:
- **CacheFirst**: Best for static assets (fonts, images)
- **NetworkFirst**: Best for dynamic content (API calls)
- **StaleWhileRevalidate**: Best for frequently updated content

### Add More Shortcuts:
Edit `public/manifest.json` ‚Üí `shortcuts` array:
```json
{
  "name": "View Services",
  "url": "/#services",
  "icons": []
}
```

## üêõ Troubleshooting

### Install button not showing?
- Ensure you're on HTTPS (localhost or deployed)
- Check manifest is valid in DevTools
- Clear browser cache and reload

### Service worker not registering?
- Check browser console for errors
- Ensure build was successful
- Verify `vite-plugin-pwa` is installed

### Icons not displaying?
- Verify icon files exist in `public/brand/`
- Check icon paths in manifest.json
- Icons must be PNG format
- Minimum size: 192x192

### App not working offline?
- Check Service Worker is active in DevTools
- Verify cache storage has assets
- Some API calls may need network
